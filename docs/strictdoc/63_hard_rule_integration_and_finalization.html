    STATEMENT: This document records the current disabled/placeholder Hard Rules state and preserves only the already agreed future direction for human-controlled Hard Rule integration and final prompt finalization.

    3. 3. Per-Agent Static Hard Rules Boundary

    STATEMENT: Several current Agent Stack JSON definitions contain a static prompt element named Hard Rules. Those instructions constrain individual agent calls. They are not the planned cross-pipeline global Hard Rules feature.

    STATEMENT: Future global Hard Rules shall remain under human control.

    5.3. Inspectable Rule Application

    STATEMENT: Future finalization shall make the applied Hard Rules inspectable and shall make visible which rules and context were included in the final prompt package.
    GHOST-HARD-RULES-FUTURE-PREVIEW-SEND-CONSISTENCY

    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-HARD-RULES-FUTURE-BEFORE-SEND Future Integration Before Final Send
    STATEMENT: Future Hard Rule finalization shall remain aligned with the authoritative engineered-prompt assembly so the inspectable preview and the prompt actually sent to the model do not silently diverge.
    GHOST-HARD-RULES-FUTURE-NO-SILENT-MUTATION
    5.5. No Silent Rule Mutation
    UID: GHOST-HARD-RULES-FUTURE-NO-SILENT-MUTATION

    STATEMENT: Future runtime processing shall not silently modify, weaken, remove, or replace human-approved global Hard Rules without an explicit governed action visible to the human authority.
