    1. 1. Purpose and Scope

    STATEMENT: This document defines the top-level requirements for GHOST Knowledge Management.

    Knowledge Management is the document-evidence context path that begins after prompt preparation and ends when document evidence has been selected, condensed, and written into the shared SuperPrompt for final prompt composition.

    Detailed ingestion, retrieval/reranking, and evidence-selection/condensation behavior is owned by child requirement files 41, 42, and 43.

    STATEMENT: GHOST shall provide a Knowledge Management capability that transforms project documents into query-relevant, traceable document context for the current request.

    The capability shall include document ingestion, document retrieval, optional reranking, semantic evidence selection, evidence condensation, and SuperPrompt write-back.

    The capability shall not own prompt preprocessing, memory retrieval, memory recording, final answer generation, GUI rendering, or future guardrail enforcement.

    GHOST-KM-CURRENT-CAPABILITIES
    1.4. Current Executable Knowledge Capabilities
    UID: GHOST-KM-CURRENT-CAPABILITIES
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-KM-ROOT Knowledge Management Scope
    STATEMENT: The current executable Knowledge Management baseline shall include:
    - project-based raw-document ingestion;
    - deterministic content hashing and manifest comparison;
    - overlapping character-based chunking;
    - dense OpenAI embeddings and persistent Chroma storage;
    - optional SPLADE sparse encoding and storage;
    - query projection and deterministic query splitting;
    - dense candidate generation;
    - optional SPLADE rescoring and weighted reciprocal-rank fusion;
    - optional ColBERT reranking and a second weighted reciprocal-rank fusion;
    - A3 usefulness classification and deterministic useful-only selection;
    - A4 three-call evidence classification and condensation;
    - document evidence write-back into SuperPrompt.

    Unimplemented knowledge graphs, cross-context overlap removal, A5, guardrails, and background synchronization agents shall not be presented as current Knowledge Management behavior.

    2. 2. Functional Architecture and Responsibility Boundaries

    STATEMENT: The conceptual GHOST context-generation architecture is:

    PreProcessing
    -> Knowledge Management and Memory Management as logically independent context paths
    -> Final Prompt Composition

    Knowledge Management contains internal stages but is one top-level document-evidence capability.

    5. 5. Independence from Memory and Future Context Coordination

    STATEMENT: Knowledge and memory context are currently generated separately and combined only in the final prompt projection.
