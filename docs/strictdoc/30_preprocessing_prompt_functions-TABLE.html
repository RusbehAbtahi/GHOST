        1. Purpose and Scope
        TEXT
        This document defines the requirements for the GHOST prompt-preparation capability from receipt of raw user text until the prepared SuperPrompt is handed to the Knowledge Management and Memory Management retrieval paths.

        The capability has three major functional parts:
        1. deterministic prompt parsing, canonical field mapping, mandatory-field handling, and initial SuperPrompt construction;
        2. ActiveBrief-aware prompt materiality and topic-relation classification followed by deterministic routing policy;
        3. A2 PromptShaper classification of system, audience, tone, depth, confidence, and evidence policy.

        This document defines requirements. It does not serve as an implementation-status report.

        GHOST-PREPROCESS-ROOT
        Preprocessing and Prompt Functions Scope
        GHOST shall provide one prompt-preparation capability that transforms raw user input into a structured and routed SuperPrompt ready for downstream Knowledge Management and Memory Management processing.

        The capability shall include deterministic preprocessing, ActiveBrief relation classification, deterministic routing decisions, A2 PromptShaper execution, synchronized retrieval-query construction, and final prompt-preview projection.

        The capability shall stop at the handoff boundary and shall not execute document retrieval, memory retrieval, document reranking, evidence selection, evidence condensation, or memory-context synthesis.

        Current Executable Scope
        The current executable scope shall include:
        - schema-driven deterministic header parsing and name resolution;
        - TASK and CONTEXT fallback behavior;
        - default handling for active mandatory fields;

        2. Functional Architecture and Responsibility Boundaries
        TEXT
        The prompt-preparation path combines deterministic processing and narrowly bounded LLM classification.

        The high-level flow is:

        raw user text
        -> deterministic parsing and canonicalization
        -> initial SuperPrompt update
        -> ActiveBrief relation classification
        -> deterministic six-state routing policy
        -> synchronized retrieval-query projection
        -> A2 PromptShaper
        -> prepared handoff to Knowledge and Memory Retrieval

        GHOST-PREPROCESS-SCHEMA-NORMALIZATION
        Header-Key Normalization
        Prompt header keys shall be normalized deterministically by:
        1. accepting string input only;
        2. applying Unicode NFKC normalization;
        3. converting to lowercase;
        4. trimming leading and trailing whitespace;
        5. collapsing internal whitespace runs; and
        6. removing punctuation from the header key used for matching.

        Header-key normalization shall not modify the section body text.
