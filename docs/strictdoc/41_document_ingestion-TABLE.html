        1. Purpose and Scope
        TEXT
        This document defines the requirements for project-based document ingestion within GHOST Knowledge Management.

        The ingestion path converts raw project files into persistent dense and optional sparse chunk indexes plus a deterministic manifest ledger.

        Document Ingestion Scope
        GHOST shall provide deterministic project-based document ingestion that scans raw source files, detects content changes, creates stable overlapping chunks, generates dense and optional sparse representations, persists both branches, and publishes the current file manifest.

        GHOST-DOC-INGEST-PRODUCT-NAME

        The product described by this document shall be named GHOST.

        Current source paths may remain under the ragstream package namespace without redefining the product name.
        REQUIREMENT

        GHOST-DOC-INGEST-KM-PARENT
        Document Ingestion Knowledge Management Parent
        Document Ingestion shall be governed by the capability boundaries, shared artifacts, configuration governance, and sibling ownership defined in 40_knowledge_management.sdoc.

        This document shall refine the ingestion sub-capability without redefining the complete Knowledge Management flow.
        REQUIREMENT
        		1.4

        Draft

        Ingestion Boundary
        Document ingestion shall prepare persistent project evidence stores.

        It shall not execute query-time retrieval, reranking, A3 selection, A4 condensation, memory ingestion, memory retrieval, or final prompt composition.
        REQUIREMENT

        		1.6

        Shared Dense/Sparse Chunk Identity
