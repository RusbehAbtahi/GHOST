        1. Purpose and Scope
        TEXT
        This document defines the requirements for the GHOST Agent Stack.

        The Agent Stack is the neutral infrastructure used by GHOST components that build structured LLM prompts, invoke an LLM provider, and parse structured responses.

        The current Agent Stack consists of:
        - AgentFactory;
        - AgentPrompt;
        - AgentPrompt helper modules;
        - LLMClient;
        - JSON agent configurations and external option catalogs.

        Concrete agents use this infrastructure but retain ownership of their own functional behavior, deterministic preprocessing, runtime payload construction, result interpretation, SuperPrompt mutation, and stage-specific fallback behavior.

        The GHOST Agent Stack shall provide reusable infrastructure for configuration-driven prompt composition, structured-output parsing, and external LLM invocation.

        GHOST-AGENT-STACK-CURRENT-COMPONENTS
        Current Agent Stack Components
        The current Agent Stack shall be organized around the following runtime components:

        1. AgentFactory for ordinary configuration resolution, external option-catalog resolution, AgentPrompt construction, and configuration-level caching.
        2. AgentPrompt for neutral prompt composition and structured-result parsing.
        3. AgentPrompt helper modules for generic text composition, configuration extraction, output-schema mapping, JSON extraction, and enum normalization.
        4. LLMClient for provider-facing LLM calls and usage metadata extraction.

        Concrete agent modules shall consume these components through explicit interfaces.

        2. Architectural Model and Responsibility Boundaries
        TEXT
        The Agent Stack separates reusable LLM infrastructure from agent-specific behavior.

        The generic interaction model is:

        Concrete agent
        -> obtain or create AgentPrompt
        -> prepare runtime payload
        -> AgentPrompt.compose(...)
        -> LLMClient.chat(...) or LLMClient.responses(...)
        -> AgentPrompt.parse(...)
        -> concrete agent deterministic processing and write-back

        GHOST-AGENT-STACK-SEPARATION-OF-CONCERNS
        Separation of Agent Responsibilities
        The Agent Stack shall enforce the following responsibility separation:

        - the concrete agent shall own domain-specific input preparation, business rules, deterministic checks, result interpretation, and state write-back;
        - AgentFactory shall own ordinary agent-configuration resolution and AgentPrompt construction;
        - AgentPrompt shall own neutral prompt composition and structured-result parsing;
        - LLMClient shall own provider-facing API invocation and provider response extraction.

        No Agent Stack component shall absorb responsibilities assigned to another layer merely to support one concrete agent.

        GHOST-AGENT-STACK-NEUTRALITY
        Agent Stack Neutrality
        AgentFactory, AgentPrompt, AgentPrompt helper modules, and LLMClient shall remain neutral with respect to concrete agents.

        They shall not contain A2-specific, ActiveBrief-specific, A3-specific, A4-specific, MemoryMerge-specific, retrieval-specific, or GUI-specific business rules.

        Generic support for selector, classifier, and synthesizer contracts shall not be treated as concrete-agent business logic.

        Stateless Per-Run Agent Stack Behavior
        AgentPrompt and LLMClient shall not retain concrete-agent business state between executions.

        Every LLM execution shall receive its complete runtime payload through explicit call arguments.

        Configuration reuse and provider-client reuse may persist across calls, but they shall not create hidden per-run decisions or hidden pipeline state.
