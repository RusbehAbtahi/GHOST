    UID: GHOST-CONTEXT-VALIDATION
    VERSION: 0.1

    1. 1. Purpose, Scope, and Current Boundary

    STATEMENT: This document records the limited context-cleaning and duplication-handling behavior that exists in the current code and separates it from a reserved future capability for validating the final combined Knowledge and Memory Context.

    The current implementation does not contain one universal deterministic cross-context validation or conflict-resolution stage.

    - the already agreed future direction for cross-context validation without inventing an implementation.

    UID: GHOST-CONTEXT-VALIDATION-NOT-STANDALONE-CURRENT-STAGE
    STATUS: Draft

    STATEMENT: The current Prompt Builder runner shall not be described as executing a separate Context Validation, Deduplication, and Conflict Resolution stage.

    Current limited behaviors occur inside existing projection, A4 condensation, and Memory Context synthesis functions.

    2. 2. Current Implemented Identifier and Rendering Behavior

    GHOST-CONTEXT-VALIDATION-CHUNK-ID-MAP
    2.1. Raw Document Chunk-ID Mapping
    UID: GHOST-CONTEXT-VALIDATION-CHUNK-ID-MAP
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    RELATIONS (Child):
    • GHOST-CONTEXT-VALIDATION-ID-NOT-SEMANTIC Identifier Mapping Is Not Semantic Deduplication
    STATEMENT: Before raw document evidence is rendered, the current SuperPromptProjector shall create a temporary mapping from stable chunk identifier to hydrated chunk object.

    If the same identifier appears more than once in base_context_chunks, only one object is addressable through that map.
    GHOST-CONTEXT-VALIDATION-ID-NOT-SEMANTIC
    2.2. Identifier Mapping Is Not Semantic Deduplication
    UID: GHOST-CONTEXT-VALIDATION-ID-NOT-SEMANTIC
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-CHUNK-ID-MAP Raw Document Chunk-ID Mapping
    STATEMENT: The temporary stable-ID mapping shall not be represented as semantic duplicate detection.

    It does not compare different chunk identifiers, document meaning, Memory Context, or cross-path overlap.
    GHOST-CONTEXT-VALIDATION-A3-NO-DUPLICATE-CONTRACT
    2.3. A3 Has No Duplicate Classification Contract
    UID: GHOST-CONTEXT-VALIDATION-A3-NO-DUPLICATE-CONTRACT
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    STATEMENT: The current A3 output contract shall not be described as producing duplicate markers, canonical duplicate references, or duplicate-removal decisions.

    The presence of A3ChunkStatus.DUPLICATED in the shared enum shall not by itself establish implemented A3 duplicate classification.
    GHOST-CONTEXT-VALIDATION-EMPTY-OMISSION
    2.4. Empty Context Omission
    UID: GHOST-CONTEXT-VALIDATION-EMPTY-OMISSION
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    STATEMENT: The current projector shall omit empty supporting-context categories and shall omit the entire Supporting Context section when all supported context categories are empty.

    This deterministic omission behavior shall not be represented as a full semantic validation stage.

    3. 3. Current Within-Path Repetition Handling

    STATEMENT: Current repetition handling is limited and occurs inside individual context paths. Some of it is implemented as LLM instructions rather than deterministic comparison logic.
    GHOST-CONTEXT-VALIDATION-A4-REPETITION
    3.1. A4 Document Repetition Reduction
    UID: GHOST-CONTEXT-VALIDATION-A4-REPETITION
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    RELATIONS (Child):
    • GHOST-CONTEXT-VALIDATION-A4-LLM-BOUNDARY A4 Repetition Handling Is LLM-Driven
    STATEMENT: The current A4 final-condenser instructions shall direct the condenser to remove overlap, repeated wording, formatting noise, and unwanted symbols from selected document evidence while staying loyal to the selected chunks.
    GHOST-CONTEXT-VALIDATION-A4-LLM-BOUNDARY
    3.2. A4 Repetition Handling Is LLM-Driven
    UID: GHOST-CONTEXT-VALIDATION-A4-LLM-BOUNDARY
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-A4-REPETITION A4 Document Repetition Reduction
    STATEMENT: A4 repetition reduction shall be described as behavior requested from the configured final-condenser agent, followed by deterministic write-back.

    It shall not be represented as a general deterministic semantic deduplication engine.
    GHOST-CONTEXT-VALIDATION-MEMORY-DUPLICATES
    3.3. Memory Evidence Repetition Reduction
    UID: GHOST-CONTEXT-VALIDATION-MEMORY-DUPLICATES
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    RELATIONS (Child):
    • GHOST-CONTEXT-VALIDATION-MEMORY-ACTIVEBRIEF-ANTI-DUP ActiveBrief as Memory Anti-Duplication Reference
    • GHOST-CONTEXT-VALIDATION-MEMORY-LLM-BOUNDARY Memory Deduplication Is Instruction-Based
    STATEMENT: The current Memory Context Synthesizer instructions shall direct the agent to combine duplicate evidence, state the same episodic/semantic point once, and omit memory evidence unrelated to the effective retrieval query.
    GHOST-CONTEXT-VALIDATION-MEMORY-ACTIVEBRIEF-ANTI-DUP
    3.4. ActiveBrief as Memory Anti-Duplication Reference
    UID: GHOST-CONTEXT-VALIDATION-MEMORY-ACTIVEBRIEF-ANTI-DUP
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-MEMORY-DUPLICATES Memory Evidence Repetition Reduction
    STATEMENT: The current Memory Context Synthesizer shall receive ActiveRetrievalBrief as orientation and anti-duplication reference and shall be instructed not to copy the brief into Memory Context or use it as a substitute for missing memory evidence.
    GHOST-CONTEXT-VALIDATION-MEMORY-LLM-BOUNDARY
    3.5. Memory Deduplication Is Instruction-Based
    UID: GHOST-CONTEXT-VALIDATION-MEMORY-LLM-BOUNDARY
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-MEMORY-DUPLICATES Memory Evidence Repetition Reduction
    STATEMENT: Current Memory Context duplicate combination and ActiveBrief anti-duplication shall be represented as LLM-agent instructions.

    They shall not be represented as a deterministic overlap detector with a defined similarity threshold or proof of complete duplicate removal.
    GHOST-CONTEXT-VALIDATION-WITHIN-PATH-ONLY
    3.6. Current Within-Path Boundary
    UID: GHOST-CONTEXT-VALIDATION-WITHIN-PATH-ONLY
    STATUS: Draft
    RELATIONS (Parent):
    • GHOST-CONTEXT-VALIDATION-ROOT Context Validation Capability Scope
    STATEMENT: Current A4 repetition reduction applies inside selected document evidence, and current Memory Context duplicate combination applies inside selected memory evidence.

    Neither behavior shall be described as comparing the final document summary against the final Memory Context.
