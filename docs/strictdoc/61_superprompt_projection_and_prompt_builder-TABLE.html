        1. Purpose, Scope, and Current Implementation Map
        TEXT
        This document defines the detailed current requirements for SuperPrompt projection, prompt-ready rendering, and the live Prompt Builder workflow through A4.

        Current implementation truth is derived from the provided source files, primarily:

        - ragstream/orchestration/super_prompt.py;
        - ragstream/orchestration/superprompt_projector.py;
        - ragstream/app/pipeline_runner.py;
        - ragstream/app/controller.py;
        - ragstream/app/ui_actions.py;
        - ragstream/app/ui_streamlit.py;
        - ragstream/retrieval/retriever_mem.py; and
        - the A2, A3, and A4 stage implementations that write into SuperPrompt.

        GHOST-SP-PROJECTION-ROOT
        SuperPrompt Projection and Prompt Builder Scope
        GHOST shall provide a projection and Prompt Builder capability that converts the current SuperPrompt state into retrieval-oriented projections, A3 comparison text, stage-visible engineered prompt previews, and the current final prompt_ready representation.

        This file shall define current implemented behavior through A4 and shall explicitly separate that behavior from the unfinished external LLM-call/send boundary, postponed A5 behavior, and future global Hard Rule integration.

        4. Retrieval Query Projection
        TEXT
        The projector currently derives retrieval query text from TASK, PURPOSE, CONTEXT, prompt materiality, topic relation, and the ActiveBrief snapshot prepared by the prompt-relation classifier.

        30_preprocessing_prompt_functions.sdoc owns the routing semantics. This section records the current projection implementation used by downstream retrieval.
        REQUIREMENT
        		4.1

        Query Projection Inputs
        Retrieval query projection shall read task, purpose, and context from SuperPrompt.body and shall read activebrief_prompt_materiality, activebrief_topic_relation, and activebrief_relation_activebrief from SuperPrompt.extras.
