        GHOST-RUN-OPS-CAPABILITY
        AWS Runtime Platform and Operations Capability
        GHOST shall provide the runtime-platform and operations capability required to reproduce the documented current AWS deployment and to add the approved initial remote MCP endpoint.

        This capability shall be limited to AWS hosting, container execution, image delivery, HTTPS routing, DNS, runtime secrets, persistent AWS runtime data, deployment activation, operational verification, and the MCP deployment boundary.

        Current Public HTTPS Request Path
        The current public request path shall be Route53 DNS to the EC2 public IPv4 address, then through the EC2 Security Group to nginx on HTTPS port 443, and from nginx to the Dockerized Streamlit backend on host port 8501.

        GHOST-RUN-TGT-MCP-TOOL-SCOPE
        Target MCP Tool Scope
        The initial MCP server shall expose only the single ghost_engineer_prompt tool defined by 73_mcp_interface.sdoc.

        That tool shall reuse the existing deterministic prompt-construction function and then the existing A2 PromptShaper on the same request-isolated SuperPrompt. PreProcessing and A2 shall not be advertised as separate MCP tools.

        The MCP server shall not expose document ingestion, document retrieval, reranking, A3, A4, memory, file management, Streamlit control, answer generation, or the full GHOST pipeline.

        GHOST-RUN-TGT-MCP-OAUTH-INTEROPERABILITY
        Target Claude and ChatGPT OAuth Interoperability
        The remote MCP endpoint shall use the MCP authorization model for HTTP transports and shall complete authenticated connection and invocation of ghost_engineer_prompt from:

        - the owner's ChatGPT web client through its custom MCP plugin capability; and
        - Claude through its remote custom connector capability.

        Acceptance shall be demonstrated against both real clients rather than inferred only from a local test client or MCP Inspector.

        GHOST-RUN-TGT-MCP-STREAMLIT-NONREGRESSION
        Target MCP Streamlit Non-Regression
        Adding or restarting the MCP container shall not replace the current Streamlit container, change the current public root application route, open port 8501 or the MCP backend port publicly, or alter the current document-ingestion bind mounts.
