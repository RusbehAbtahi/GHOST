        1. Purpose, Scope, and Current Boundary
        TEXT
        This document records the limited context-cleaning and duplication-handling behavior that exists in the current code and separates it from a reserved future capability for validating the final combined Knowledge and Memory Context.

        The current implementation does not contain one universal deterministic cross-context validation or conflict-resolution stage.

        GHOST-CONTEXT-VALIDATION-ROOT
        Context Validation Capability Scope
        This file shall define:

        - current identifier-level and within-path repetition-handling facts;
        - current LLM-instruction-based deduplication behavior inside A4 and Memory Context synthesis;
        - the explicit absence of a current universal cross-path validator or conflict resolver; and
        - the already agreed future direction for cross-context validation without inventing an implementation.
        REQUIREMENT
        		1.2

        GHOST-CONTEXT-VALIDATION-CHUNK-ID-MAP
        Raw Document Chunk-ID Mapping
        Before raw document evidence is rendered, the current SuperPromptProjector shall create a temporary mapping from stable chunk identifier to hydrated chunk object.

        If the same identifier appears more than once in base_context_chunks, only one object is addressable through that map.

        GHOST-CONTEXT-VALIDATION-ID-NOT-SEMANTIC
        Identifier Mapping Is Not Semantic Deduplication
        The temporary stable-ID mapping shall not be represented as semantic duplicate detection.

        It does not compare different chunk identifiers, document meaning, Memory Context, or cross-path overlap.

        Empty Context Omission
        The current projector shall omit empty supporting-context categories and shall omit the entire Supporting Context section when all supported context categories are empty.

        This deterministic omission behavior shall not be represented as a full semantic validation stage.

        The current Memory Context Synthesizer shall receive ActiveRetrievalBrief as orientation and anti-duplication reference and shall be instructed not to copy the brief into Memory Context or use it as a substitute for missing memory evidence.

        Memory Deduplication Is Instruction-Based
        Current Memory Context duplicate combination and ActiveBrief anti-duplication shall be represented as LLM-agent instructions.

        They shall not be represented as a deterministic overlap detector with a defined similarity threshold or proof of complete duplicate removal.

        GHOST-CONTEXT-VALIDATION-WITHIN-PATH-ONLY
        Current Within-Path Boundary
        Current A4 repetition reduction applies inside selected document evidence, and current Memory Context duplicate combination applies inside selected memory evidence.

        Neither behavior shall be described as comparing the final document summary against the final Memory Context.
